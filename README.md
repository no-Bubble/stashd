# 📦 Stashd Cloud Storage

This repository serves as the decentralized backend for **Stashd**, the NFC-first inventory management system. It manages group synchronization, real-time updates, and global recipe sharing via the GitHub Contents API.

---

## 🏗️ Repository Architecture

The database follows a flat-file, folder-based structure optimized for low-latency synchronization and group-based access control.

### 📂 Root Directory
- `/recipes/` — **Public Global Recipes**: AI-generated recipes available to all Stashd users worldwide.
- `/{groupId}/` — **Group-Specific Data**: Isolated environments for household or shared groups.

### 📂 Group Structure (`/{groupId}/`)
- `group.json` — **Member Registry**: Admin/Member roles and device installation IDs.
- `shopping_list.json` — **Live Shopping List**: Shared grocery list for the household.
- `barcodes.json` — (Legacy) Full barcode lookup table.
- `/barcodes/` — **Atomic Barcode Store**: Individual `{uuid}.json` files for high-concurrency lookup.
- `/recipes/` — **Private Recipes**: Group-specific recipes shared only with household members.
- `/temp/` — **Next-Scan Queue**: Temporary storage for "Add to next scanned tag" operations.
- `/live/`
  - `queue.txt` — **Real-time Mutation Bus**: A simple text-based line bus for signaling updates (shelf changes, recipe generation, etc.) without hitting GitHub API limits.

### 📂 Shelf Store (`/{groupId}/shelves/`)
- `/{tagId}/shelf.json` — **Shelf Metadata**: Icon, name, and general settings for a physical shelf/container.
- `/{tagId}/{uuid}.json` — **Inventory Items**: Atomic storage for items on this specific shelf.

## 🔄 Synchronization Mechanism

### 1. The Real-time Mutation Bus
Stashd uses a "Wait-then-Cleanup" polling strategy to ensure 5-second synchronization between devices without wasting GitHub API tokens:
1. **Mutation**: A device appends `type id timestamp` to `${groupId}/live/queue.txt`.
2. **Polling**: Other devices read the raw text from `raw.githubusercontent.com` every 5 seconds.
3. **Trigger**: If new lines are detected, the device fetches the corresponding data update.
4. **Cleanup**: 5 seconds after processing, a device removes its processed line from the queue API.

### 2. Physical-to-Cloud Bridge
Data is primarily stored on physical NFC tags. The cloud acts as a **persistent mirror** for:
- Remote inventory viewing.
- Synchronized shopping lists.
- Collaborative recipe generation.

## 🔒 Privacy & Access
Data is managed via personal access tokens. While the repository structure is public, data within group folders is only accessible to those who possess the group's 6-character cryptographic code.
